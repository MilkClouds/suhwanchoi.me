01
01 The Problem

Per-benchmark integration doesn't scale

VLA models are evaluated on many simulation benchmarks — but wiring each one in is heavy, repeated engineering.

  • Duplicated effort. Every model repo keeps its own eval scripts, drifting apart silently.
  • Dependency conflicts. LIBERO needs Python 3.8, ManiSkill2 needs 3.10, CALVIN needs PyBullet — no shared environment.
  • Underspecified protocols. Seeds, episode counts and preprocessing are omitted from papers.
O(N×M)
Naïve cost — re-integrate for every model × benchmark pair.
vla-eval
O(N+M)
Integrate each once — the full N×M matrix then fills itself.
02
02 Framework Design

Decouple the model from the benchmark

Inspired by the decoupled design of lm-evaluation-harness for LLMs, vla-eval splits model inference from benchmark execution over a WebSocket + msgpack protocol — and isolates each benchmark inside its own Docker container.

  • Models integrate once. One blocking predict() — ~50 lines.
  • Benchmarks integrate once. Four methods inside a pinned Docker image.
model side, in fullclass OpenVLAServer(PredictModelServer):
  def predict(self, obs, ctx):
    img = Image.fromarray(
      next(iter(obs["images"].values())))
    prompt = f"In: action for {obs['task']}? Out:"
    inp = self._proc(prompt, img).to("cuda")
    act = self._model.predict_action(**inp)
    return {"actions": act}

04
04 Validation — pitfalls

One hidden parameter ⇒ catastrophic swings

Each reproduction unmasked an undocumented setting that, alone, moves success rate by tens of points.

  • X-VLA — wrong proprioceptive state source on LIBERO: 97.8% → 42% (a 55 pp swing).
  • OpenVLA-OFT — quaternion antipodal normalization mismatch: LIBERO-Long 95% → 56%.
  • GR00T — missing end-effector pose field on SimplerEnv: 30–55% → 0%.
  • OpenVLA — undocumented eval-time center crop (scale 0.9): −3 pp.

vla-eval records the full configuration with every result — making each run reproducible from one file.

Cross-benchmark comparison is rare.

81% of models are evaluated on a single benchmark; only 6% on three or more.

vla-eval makes cross-benchmark comparison practical — letting the community see general capability across diverse environments and embodiments.
